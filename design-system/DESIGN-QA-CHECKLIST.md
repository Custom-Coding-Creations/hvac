# Design QA Checklist v1

## Token Compliance
- [ ] Color, spacing, type, radius, and motion use token variables only.
- [ ] No one-off hard-coded values unless approved and logged.

## Component States
- [ ] Header states validated.
- [ ] Nav states validated.
- [ ] CTA states validated.
- [ ] Form states validated.
- [ ] FAQ states validated.

## Accessibility
- [ ] Contrast matrix checks pass for all text/button pairs.
- [ ] Keyboard navigation works for nav, forms, accordions, and CTA controls.
- [ ] Labels and error messages are announced correctly.

## Responsive
- [ ] 360px, 768px, 1024px, 1440px layouts verified.
- [ ] Navigation and trust modules remain usable and readable.
- [ ] CTAs remain visible and tappable on mobile.

## Fidelity
- [ ] Template compositions align to wireframe specs.
- [ ] Conversion entry points per template are present and ordered.
